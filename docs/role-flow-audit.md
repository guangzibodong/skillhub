# SkillHub Role Flow Audit

This matrix tracks the full-flow contract for launch readiness. It is intentionally tied to concrete routes, server actions, gateway endpoints, tables, and tests so a flow cannot look complete in the UI while still depending on demo data or an unguarded backend path.

Status legend:

- Live: implemented through UI, gateway, database writes, authorization, and at least source or smoke coverage.
- Partial: implemented but missing one launch-readiness requirement such as a browser smoke, a database invariant, or an explicit empty/unavailable state.
- Blocked: the user path cannot finish against live data.
- Demo guarded: fallback data is allowed in local development, but production must return live, empty, or unavailable state.

## Demo Fallback Policy

Development may show bundled demo records when `DATABASE_URL` is absent. Production acceptance must not use demo data as a business decision source. In production, gateway and web helpers must prefer one of these states:

- Live: data is read from the database or an external provider.
- Empty: the database query succeeded and returned no records.
- Unavailable: the dependency is missing or down, and the UI blocks the action with a next step.

`SKILLHUB_ENABLE_DEMO_FALLBACK=true` is a launch-readiness blocker in production. The gateway launch-readiness check already reports this as `demo_fallback`.

## Required Verification For New Flow Work

Every new or changed flow must update the matching row in this audit, then add or update:

- Gateway route and authorization scope.
- Web data helper or server action.
- TypeScript types used by the UI.
- Migration or query for any persisted state.
- Audit log write for money, permissions, review, payout, API key, notification, webhook, identity, or admin actions.
- Targeted test plus the relevant P0 smoke.
